Pocket Dice Xcode App Prototype

For this term-long project, I was tasked with creating a small functional app prototype within the Apple XCode framework, using a combination of the Storyboard development environemnt and some custom swift classes & programming.

For this assignment, I decided to create a life tracker app. As an avid Magic the Gathering player, I thought it would be a fun challenge to create my own life tracking app in order to manage health when playing games with my friends. My goals to make this app possible were to:

  • Design all my screens, then create responsive screen size versions of them in Xcode using the storyboard editor.
  • Implement my app-defining mechanic, being that a user could tap above their unique life total to increase it and below to subtract from it.
  • Pass data between screens so that users can set the defualt starting life for their game to unique values.

PROCESS & INSIGHT

Part I

Creating my Screens

My first step of this assignment was creating all the screens present in my application. I did a brief design mockup using Figma then began implementing my designs in XCode. My final screens included my Home page, my Life selection page, the Tracker page, and a Help page. All of theses screens were unique in their content and design and I used a combination of auto and custom contraint rules to make sure everything was responsive and fit correctly on any phone screen size.

Implementing my Core Functionality

My next step was to create dynamic button functionality on the main tracker page that allowed for addition and subtraction to all life totals depending on where a user clicked. This was the main purpose of my app so there was little room to compromise on my vision here. Before beginning coding, I polished up the screen, making sure to set up 8 invisible button touch spaces, (two per life counter box) in order for each player to have the functionality to manipulate their specific life total.

I then intially created a custom controller class for the screen. Within this code space I did a couple different things to hook up my app on the backend. First, I hooked up all the life total text labels that were displayed to the players so that I could manipulate them in the future. I then made button eventlisteners for all 8 of the buttons present on the screen (two for each user, one minus button below their life total and one plus button above it). Next I programmed two functions that added and subtracted life totals and hooked them up to each button event listener. It was by doing this that each of the 8 buttons added and subtracted life from their unique player label.

Passing my Data

The last main step of my objectives with this app prototype was to pass starting life data from my life selection page to my tracker page. The goal of this task was to give players different starting life options in case they wanted shorter or longer games. in order to implement this feature, I set up a segue function that sent different integer values (depending on which of the three buttons was clicked) to variables I defined on the next screen (the Tracker screen). Then once on the tracker screen, I slotted those variable values into the text life total labels. It was by doing this, that I was able to intialize game starting totals with different values.
