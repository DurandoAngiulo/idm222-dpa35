Epic Games Store Microinteraction Redesign

For this term-long project, I was assigned to create a complex multi-level microinteraction using HTML, CSS, and Javascript to improve upon an existing microinteraction on a live site. I chose the Epic Games Store page as my site and decided to improve upon the game list section(where all the games are listed for purchase). Currently, when you click on a game, the site takes you to a new page with all the game information on it.

My goal was to replace this simple interaction with a complex three-tiered microinteraction:

  • Firstly, I wanted to implement a hover state so that when a user hovers on a game, it grows slightly with an information box fading in over it from the bottom.
  • Secondly, to create hover states on certain icons of the information box that grow and change opacity when hovering over them.
  • And finally, on clicking one of these icons (an info button), to fade in a bigger modal on top of the the page with a reduced store page for the game where a user can get more information and buy/add it to their cart.

PROCESS & INSIGHT

Part I

Alpha Build

When I started working on this project, I mainly broke it down and completed it in pieces, starting with my Alpha Build. This build included creating a successful hover state on one image so that it grew 20% bigger and its info box came up inside it. Before I even started coding anything, I made a brief design system using Epic’s site typography and colors and wireframed out my animation flow in figma to keep myself on track as I started development.

Next I began coding my first iteration of my microinteraction. The HTML and CSS didn’t take long and I also used Bootstrap along the way to make my styling go faster. When it came time to actually animating everything, I used a plug-in called anime.js in my Javascript to implement the clean expand effect I was looking for in my image hover state. After finishing my Alpha Build, I made some additional edits to my animation timing based on feedback I received from my class critique.

Beta Build

For my beta build, I mainly expanded on my alpha by giving every image a hover state. I also created my second level microinteraction in this build, populating my fade-in text boxes with information about their respective game, and with certain elements that on hover would also expand and change opacity.

What I struggled most with this time around was using Javascript to correctly chain all my hover animations to the same function. Doing some research, I ended up using For Each loops in my event listeners as well as element.children to overcome this challenge. Once again, during class critique I received feedback from my peers and professor and made some minor tweaks to the look and feel of my microinteractions.

FINAL MICROINTERACTION PROTOTYPE

Part II

Final Build

To finish off my microinteraction for my final build, I added an interactable modal that appears when a user clicks on the info button that shows up when a game image is hovered on. This modal is complete with a more detailed description about the game, other hoverable buttons such as “Add to Cart” & “Buy Now”, and an image that turns into a gif when hovered on. To exit this modal, users simply need to click on the X button in the top right corner and the interaction will revert back to its start state.

Project Reflection

I am very proud of how my final interaction turned out. During the course of this project I learned many new skills and successfully persevered when things got tough. And while there are still some future things i'd like to fix for this microinteraction such as the X button spacing and image to Gif hover lag, in the end I feel that I accomplished my goal and significantly improved upon the original interaction present on the Epic Games Store Page.
