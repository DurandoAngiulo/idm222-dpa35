Game Nexus: Videogame Ecommerce Website

For this term-long project, I was assigned to create an Ecommerce web application protoype using the React framework that included a live database, visitable product pages, a functional shopping cart, and a interactable admin page. I chose to make a videogame ecommerce site for this assignment

PROCESS & INSIGHT

Part I

Laying out my Designs

Before I wrote a single line of code, I first took a moment to create some preliminary designs of what I wanted my final site to look like. While I didn't follow these mockups perfectly, the typography, colors, and layouts I put togther made the intial styling of my site go much faster.

I also took this time to gather every asset I needed for the final prototype as well as roughly planning out the type of code structures/functions I needed to build the entire web application.

CODING & DEVELOPMENT

Part II

Inital Site Construction

My development process was very iterative and I found myself designing alongside adding functionality as my siteâ€™s vision evolved.

Initially I created and stylized all my relative pages using CSS & Bootstrap. Along the way I implemented my first React components for specific elements on my pages to avoid redundant code.

Database Setup

My first stretch of development revolved around setting up a live database to pull all my game data from. I used Google Firebase to accomplish this and set up a local store in my application to make it easy to call my data wherever I needed it.

Home & Game Page Development

I then set up my home page shop and specific game pages. Usually route parameters, I passed down props through my components to link up all my game assets so that everything from imagry to text fields were actually dispayed in real time from my database to my site.

Admin Page Development

Next I focused on the creation of my backend Admin page. Using forms and a model system, I created an Admin page where all my game data could be altered in real time, from changing things like the name & description, to the price itself.

Cart Page Development

Finally, I implemented a cart system to faciliate the process of checking out a game from the site. From the cart, a user could add or remvove items, or increase the quanity of items purchased. The totals and tax would be calcuated automatically and would be shown to users before "checking out".
